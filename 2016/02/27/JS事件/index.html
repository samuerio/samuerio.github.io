JS事件

事件监听程序

JS原生的添加事件监听程序的方式有两种，一种是简单粗暴地对属性赋值，另外一种是调用函数addEventListener。

  1. DOM.onclick
  2. DOM.addEventListener([String] eventType,[EventListener|Function] listener,[boolean] useCapture = false)第三个参数决定调用监听程序的阶段「捕获阶段/冒泡阶段」,默认为「冒泡阶段」。

相对第一种方式而言，第二种方式更加灵活：可以添加多个处理程序，而且也可以移除事件处理程序。

事件对象

当事件发生,调用监听程序时,浏览器默认会对事件监听程序注入事件对象event。让我们开始了解event常用的属性和方法。

常用属性

  1. type : [String] eventType eg. click，mouseover…
  2. currentTarget : [Object] 指代当前正在处理事件的DOM元素，必定等同于this
  3. target : [Object] 指代事件的目标，不一定等同于this

由于事件是存在冒泡机制的，点击button按钮，随着点击事件的冒泡，最终会被body标签捕获，此时事件对象的target是所点击的按钮对象，而currentTarget则为当前处理事件的元素即body。

常用方法

  1. stopPropagation : 阻止事件的冒泡。注：如果只想事件在某个标签上生效，最好调用该方法阻止其冒泡，防止事件污染。
  2. preventDefault : 阻止默认元素的默认行为
