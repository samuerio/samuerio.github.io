Shell下的权限控制

发表于 2016-03-03

用户授权（sudo）

若当前用户没有执行某命令的权限可以使用sudo给其授权

文件/目录授权（chmod）

权限 r w x -
含义 read write execute 相应权限未被授予

查看权限

profile文件，权限信息分析：

-rwxrwxr-x

  1. 首位- : 类型
  2. rwx : User
  3. rwx : Group
  4. r—x : Other

改变权限

  • 方式一：

u「user」g「group」o「other」a「all」

eg.

1. chmod -r file : 删除所有用户对file的`r`权限
2. chmod u +r file : 增加所有用户对file的`r`权限
3. chmod =r file : 所有用户对file只有`r`权限
  • 方式二:
权限组合 二进制数 十进制数
可读可写可执行 rwx 111 7
可读可执行 r-x 101 5
可读可写 rw- 110 6

eg.

1. chmod 777 file : 所有用户对file有rwx权限
2. chmod 175 file : file创建者对file有x权限，group对file有rwx权限，othor对file有rx权限

JS事件监听

发表于 2016-02-27

事件监听程序

JS原生的添加事件监听程序的方式有两种，一种是简单粗暴地对属性赋值，另外一种是调用函数addEventListener。

  1. DOM.onclick
  2. DOM.addEventListener([String] eventType,[EventListener|Function] listener,[boolean] useCapture = false)第三个参数决定调用监听程序的阶段「捕获阶段/冒泡阶段」,默认为「冒泡阶段」。

相对第一种方式而言，第二种方式更加灵活：可以添加多个处理程序，而且也可以移除事件处理程序。
