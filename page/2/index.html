Shell下的文件和路径

路径

  • 改变路径：cd /（/为根目录）
  • 浏览路径下的文件：ls -l /home(-l是以列表形式显示）
  • 显示当前工作路径：pwd
  • 创造目录：mkdir -p(-p代表创造多级目录）
  • 删除目录：rmdir

文件

  • 删除： rm -rf「强制删除」
  • 类型： file「显示文件类型」
  • 新建： touch「新建文件或更新时间戳」
  • 复制： cp filea fileb「当前目录多了fileb」
  • 重命名/移动： mv filea fileb
  • 查看： cat more less head tail「控制台查看文本内容，但无法编辑」

Vi

概述：

  • 进入编辑环境: vi
  • 打开/创建: vi file

3种模式：

  1. 命令模式

    跳行： G「跳到末行」 nG「跳到n行」
    复制： yy「复制当前整行」 nyy「复制从当前行开始的n行」
    黏贴： p「插入的形式，以当前行的下一行作为插入点」
    删除： dd「删除当前行」 ndd「删除从当前行开始的n行」
    翻页： ctrl+f「向后翻整页」 ctrl+b「向前翻整页」
    撤销： u「取消最近一次的操作」U「取消对当前进行的所有操作」
    恢复： ctrl+r「对撤销进行恢复」
    查询： /word（自上而下查找） ?word（自下而上查找） n（定位下一个）N（定位上一个)

  2. 输入模式

  3. 末行模式

    替换： %s/old/new/g「替换文件范围内的所有“old”为“new”」
    行号： set nu「设置」 set nonu「取消」
    退出： q!「强制退出」
    保存： wq「保存并退出」 w filename「另存为filename」
    追加： r filename「追加filename内容到当前文件中」
    打开： e！filename「打开filename，并放弃当前文件的修改」 e filename「打开filename」
    帮助： help

JS原型链实现继承

ECMAScript只支持实现继承，不支持接口继承，并将原型链作为其主要的实现方式。

每个对象上的proto属性都保存有原型的引用。A继承B，只要实现A原型(A.prototype)的原型为B原型(B.prototype)即:A.prototype.__proto__ = B.prototype。从这里可以看出，要实现继承只要实现原型链即可。

负margin与absolute、relative

负margin

参考线是保持不动的

margin-top、margin-left是以外元素为参考线，margin-right、margin-bottom则以元素本身为参考线。

负margin在一定程度上可以替代「relative」「absolute」达到一定的布局效果。并且负margin始终是处于「标准文档流」中的，对整体布局影响小。

relative和absolute

relative

  • 自身位置为基准
  • 处于标准文档流

absolute

  • 以包含块为基准
  • 脱离标准文档流

注意：若absolute未设置偏移量，则无论是否存在已定位的祖先元素，都保持在元素最初的位置。
